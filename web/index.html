AUTOMO runs in this browser but thinks on your computer. The model never leaves your machine — this page just talks to it over Local Network Access.

Start Ollama, allowing this page
So your model accepts requests from this origin (CORS).
copyOLLAMA_ORIGINS='ORIGIN' ollama serve
2
Pull a model (once) 
copyollama pull llama3.2
3
Connect & grant the prompt
Chrome will ask to allow local-network access. Click Allow.

The browser triggers /api/pull on your machine — no terminal needed. (Setting the origin + starting Ollama still has to happen once on the machine — the browser can't launch a process.)
folder: none
Grants the agent fs_list · fs_read · fs_write on a folder you pick (File System Access). It always has private mem_* tools on OPFS. The model calls these itself — try "list the markdown files and summarize the README".
Streamable HTTP connects directly (LNA-gated for local; auth + headers supported). stdio runs the command through the bridge daemon over LNA. Tools become the agent's tools (prefixed mcp_<label>_). Hosted MCP tools aren't listed — they route through OpenAI's Responses API and need an OpenAI model, not a local one.
The LLM context. Sent as instructions on every turn, with live run context appended (model, granted folder, MCP servers, date). Blank = the built-in default.
History is trimmed to this size before each turn (oldest messages drop first, tool call/output pairs kept together) so long chats + tool loops don't overflow the model.
